<objective>
Give operators legacy `config__*` env-var access to the fork describe toggles without breaking the
existing `PR_DESCRIPTION__*` env vars or `[pr_description]` TOML overrides. Add a dual-read helper
(`config.*` first, `pr_description.*` fallback), a load-time mirror so `CONFIG__USE_DESCRIPTION_MARKERS`
reaches upstream read sites without editing them, and the new `enable_pr_agent_output` toggle key
(Plan 02 consumes it). Keys stay defined in `[pr_description]`; byte-identical-when-off preserved.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Operator env vars -> settings | `CONFIG__*` / `PR_DESCRIPTION__*` are operator-controlled; dynaconf coerces boolean literals |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-04-01 | Tampering | Env var coercion | low | accept | dynaconf coerces `true`/`false` literals to bool; integer `1`/`0` parse as int (still truthy). Documented in fork docs; .get(key, False) read pattern keeps defaults safe. |
| T-04-02 | Elevation | Mirror clobbers explicit pr_description value | low | mitigate | Mirror writes only when config.use_description_markers is explicitly non-None; never overwrites an explicit pr_description value with a default. |

</threat_model>
