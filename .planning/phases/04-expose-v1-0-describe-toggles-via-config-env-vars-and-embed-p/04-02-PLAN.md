<objective>
Make the org template a single org-formatted body: add a `## Changes` section carrying the file
walkthrough + mermaid diagram (filled from the raw walkthrough table and the sanitized diagram),
and honor `enable_pr_agent_output` (default false) so PR-Agent's own default description body is
suppressed when the org template is active. Walkthrough/diagram are COPY-embedded (render in the
template; also in the default body when the operator opts back in). Byte-identical when the org
template is off.
</objective>

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-04-03 | Tampering | Diagram backreference injection | low | mitigate | Use a lambda replacement (lambda m: diagram or "") instead of a raw re.sub replacement string so backreferences / backslashes in mermaid content are not interpreted |
| T-04-04 | Information Disclosure | Walkthrough leaks into suppressed body | low | mitigate | Suppression returns only the block; clean_body is never appended when enable_pr_agent_output is false |
| T-04-05 | Denial of Service | process_pr_files_prediction raises | low | mitigate | try/except around the raw-table call in _stash_org_template_fields; fall back to "" and log warning — marker fills empty, describe never crashes |

</threat_model>
