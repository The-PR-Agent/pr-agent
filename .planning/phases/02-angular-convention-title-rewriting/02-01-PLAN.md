<objective>
Add the pure Python validator `_normalize_angular_title(title: str) -> str | None` to `pr_agent/tools/pr_description.py` — the heart of Phase 2. It validates an AI-produced MR title against the Angular Commit Convention and repairs common defects (synonym type mapping, lowercase-first-char, strip single trailing period, drop empty/invalid scope parens, word-boundary truncate over 70 chars), returning a conforming `type(scope): summary` string or `None` when the output cannot be salvaged.

Purpose: This is the salvage-and-fallback engine that guarantees GitLab only ever receives a valid Angular title or nothing at all (TITLE-02..06). It is a self-contained pure function, unit-tested in isolation with the 25-row adversarial fixture corpus from RESEARCH.md section 3 — the prime TDD candidate for this phase.

Output: `_normalize_angular_title` in `pr_description.py` (plus module-level constants) and `tests/unittest/test_normalize_angular_title.py`.
</objective>
