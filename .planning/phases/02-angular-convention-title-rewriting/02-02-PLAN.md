## Phase Goal

**As a** GitLab MR author using `describe`, **I want to** enable a single `enable_conventional_title` toggle, **so that** my MR title is automatically rewritten to a valid Angular `type(scope): summary` and published — with no manual formatting and without also having to set `generate_ai_title`.

<objective>
Wire the Phase 2 vertical slice end-to-end so the validated Angular title actually reaches GitLab. Three connected changes in `pr_agent/tools/pr_description.py`, all gated on `enable_conventional_title`:
1. CFG-04 auto-force: capture the AI title into `self.ai_title` before `_prepare_pr_answer*` consumes it, then widen the publish-seam conditional (~line 210-213) so enabling `enable_conventional_title` alone routes the AI title through `_normalize_angular_title` and on to `publish_description` — without requiring `generate_ai_title=true`.
2. TITLE-01 steering: append an Angular-format instruction block to the effective `extra_instructions` in `__init__` when the toggle is on (no inline edits to `pr_description_prompts.toml`).
3. Fallback safety: when the validator returns `None`, pass `None` (never `""`) to `publish_description` so the existing GitLab provider path leaves the MR title untouched.

Purpose: This is the seam that proves the whole feature — without it the validator from Plan 01 never affects a real MR. It closes CFG-04 and TITLE-01 and locks in the byte-identical-when-off guarantee (success criterion #5).
Output: Three guarded edits to `pr_description.py` plus `tests/unittest/test_conventional_title_publish_seam.py`.
</objective>
