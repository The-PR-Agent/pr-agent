<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| AI YAML output -> rendered MR body | `what_why` and `note_risk` are untrusted model output inserted into markdown |
| Existing MR description -> checkbox states | Existing description may contain arbitrary user markdown but only checklist states are preserved |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-01 | Tampering | Sentinel replacement | medium | mitigate | Replace only bounded sentinel block with non-greedy regex and strip any accidental existing block from fresh body |
| T-03-02 | Tampering | Checklist preservation | low | mitigate | Preserve only `[x]`/`[ ]` state for matching checklist labels; do not copy arbitrary old block text |
| T-03-03 | Denial of Service | Missing template file | low | mitigate | Reuse `load_org_template()` graceful fallback and skip prepend on empty template |
</threat_model>
