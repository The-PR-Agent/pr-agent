<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| filesystem -> process | `org_template.md` is read from a fixed package-relative path derived from `__file__`; no user/network input crosses this boundary |
| config -> process | Two boolean toggles read from TOML/env; values are booleans, not executable |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-01 | Tampering | `_ORG_TEMPLATE_PATH` file read | low | accept | Path is a fixed constant under the package (`Path(__file__).parent.parent / "settings" / "org_template.md"`); no user input concatenated, so no path traversal. File ships in the repo. |
| T-01-02 | Information Disclosure | template content | low | accept | Template is a static, non-secret markdown scaffold committed to the fork; contains no credentials or PII. |
| T-01-03 | Elevation of Privilege | toggle defaults | low | mitigate | Both toggles ship `false`; loader is inert (unwired), so no new behavior is reachable this phase. |
</threat_model>
